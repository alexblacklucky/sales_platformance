# CLAUDE.md — PLATFORMANCE / sales_platformance

Память проекта для Claude Code. Читается автоматически в начале каждой сессии.
**Прежде чем делать аналитику — открой `ANALYTICS_GUIDE.md` (там вся логика и история).**

## Что это за проект
Статический сайт-база знаний отдела продаж PLATFORMANCE (агентство по продвижению
на маркетплейсах Avito / WB / OZON). Чистый HTML/CSS/JS, без сборки и бэкенда.
Живой сайт: **https://alexblacklucky.github.io** (GitHub Pages).

## Главные правила (не нарушать)
- **Деплой:** GitHub Pages отдаётся из ветки **`main`**. Изменения видны на сайте
  ТОЛЬКО после влития в `main` (PR → merge). Прямой push в `main` блокируется —
  лить через Pull Request.
- **Рабочая ветка:** `claude/repo-exploration-lb95di`.
- После merge: подождать ~1–2 мин и обновить сайт жёстко `Ctrl+Shift+R`.
- Стиль всех дашбордов — светлая фиолетовая тема, шаблон `analytics/june.html`.

## Структура
- `index.html` — лаунчер/роутер + окна аналитики, инжект секции «Аналитика» в сайдбар.
- `analytics/<month>.html` — Avito Dashboard за месяц (`may.html`, `june.html`).
- `analytics/bitrix-<month>.html` — Bitrix Dashboard (воронка) за месяц.
- `analytics/overview.html` — «Общая аналитика» (сводка Avito + Bitrix).
- `ANALYTICS_GUIDE.md` — подробная инструкция: расчёты, форматы, история данных.

## Напоминания по расчётам (кратко, детали — в гайде)
- Bitrix `TOTAL` = сумма ВСЕХ колонок канбана; `MEET` = сумма 7 колонок
  (презентация, надо подумать, моросят, договор/счёт, провалена, БОМЖИК 2, успешна);
  `WON` = ручное число успешных сделок от заказчика (НЕ значение колонки).
- Avito «Средняя конверсия в контакт» = контакты ÷ просмотры (вручную, с пометкой).
