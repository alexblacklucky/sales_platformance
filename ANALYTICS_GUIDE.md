# Инструкция по аналитике PLATFORMANCE

> Этот файл — «память» проекта. Перед добавлением аналитики за новый месяц
> прочитай его целиком. Здесь зафиксированы: структура сайта, форматы отчётов,
> точная логика расчётов и исторические данные. Цель — чтобы любой новый месяц
> (например, июль) собирался **в точности по той же схеме**.

---

## 1. Структура репозитория

| Файл | Назначение |
|------|------------|
| `index.html` | Лаунчер/роутер сайта (SPA-оболочка). Грузит базу знаний из закреплённого коммита, инжектит секцию «Аналитика» в сайдбар, содержит окна аналитики. |
| `analytics/<month>.html` | **Avito Dashboard** за месяц. Пример: `may.html`, `june.html`. |
| `analytics/bitrix-<month>.html` | **Bitrix Dashboard** (воронка продаж) за месяц. Пример: `bitrix-may.html`, `bitrix-june.html`. |
| `analytics/overview.html` | **Общая аналитика** — сводка Avito + Bitrix по всем месяцам. |
| `База_знаний_..._презентации.html` | Исходник базы знаний (тяжёлый, со встроенными картинками). |

Месяцы по-английски в именах файлов: `may, june, july, august, september, october, november, december`.

---

## 2. Где что встраивается (навигация)

- В сайдбаре базы знаний есть большая секция **«Аналитика»** (создаётся инжектом в `injectAnalyticsLink()` внутри `index.html`). В ней три пункта:
  - **Аналитика Avito** → окно `analytics-view`, помесячный бар, грузит `analytics/<month>.html`
  - **Bitrix Dashboard** → окно `bitrix-view`, помесячный бар, грузит `analytics/bitrix-<month>.html`
  - **Общая аналитика** → окно `overview-view`, грузит `analytics/overview.html`
- Роуты (hash): `#analytics-<month>`, `#bitrix-<month>`, `#overview`.
- Помесячные бары в `index.html`: у Avito кнопки `data-month="..."`, у Bitrix `data-bmonth="..."`. Доступный месяц помечается классом `available` (зелёная точка).

## 3. Как добавить новый месяц (чек-лист, на примере ИЮЛЯ)

1. **Avito:** скопировать `analytics/june.html` → `analytics/july.html`. Заменить:
   - `<title>` и `<h1>` → «Июль 2026», текст периода → «1–31 июля», `<footer>` → «Июль 2026».
   - заголовок «Общая сводка за **июль**».
   - массивы `summary`, `profiles`, `recs` — новыми данными (формат ниже).
2. **Bitrix:** скопировать `analytics/bitrix-june.html` → `analytics/bitrix-july.html`. Заменить:
   - `<title>`, `<h1>`, `<footer>` → «Июль 2026».
   - константы `TOTAL`, `MEET`, `WON` (логика расчёта — раздел 5).
   - массив `stages` (все стадии канбана), `recs`.
3. **index.html:** пометить июль доступным — добавить класс `available`:
   - в Avito-баре: `data-month="july"`
   - в Bitrix-баре: `data-bmonth="july"`
4. **overview.html:** добавить столбец/данные за июль (см. раздел 6).
5. **Залить:** коммит → PR в `main` → merge (Pages отдаётся из ветки `main`!).
6. Проверить на сайте с жёстким обновлением `Ctrl+Shift+R`.

**Стиль:** все дашборды — светлая фиолетовая тема. Канонический шаблон — `june.html`. Тёмную тему НЕ использовать.

## 4. Формат Avito Dashboard (источник: скриншоты личных кабинетов Avito)

Данные в `<script>` тремя массивами:

- `summary` — карточки сводки: `['Подпись','Значение']` или `['Подпись','Значение','примечание']`. Состав:
  Профилей в работе · Суммарные просмотры · Суммарные контакты · Суммарные расходы ·
  Средняя цена за просмотр · Средняя цена за контакт · Средняя конверсия в контакт ·
  Суммарное избранное · Активных объявлений всего.
- `profiles` — по каждому профилю:
  `{name, badge:'Контакты ±X%', tone:'up|down|flat',
    m:[['значение','Просмотры','±%','up|down|flat'], [...Контакты...], [...Избранное...]],
    e:[['Конверсия','X%'],['Цена за просмотр','X ₽'],['Цена за контакт','X ₽'],['Расходы','X ₽ (±%)'],['Активные объявления','X']]}`
- `recs` — массив строк-выводов (можно с `<b>…</b>`).

**ВАЖНО — «Средняя конверсия в контакт»:** считается ВРУЧНУЮ как
`суммарные контакты ÷ суммарные просмотры` (НЕ среднее из конверсий профилей).
В подписи карточки это обязательно указывать, например:
`'223 / 3 017 — посчитано вручную (контакты ÷ просмотры), а не среднее по профилям'`.

Десятичные — через запятую, тысячи — через пробел (`102 737,5 ₽`).

## 5. Формат Bitrix Dashboard (источник: канбан сделок в Битрикс24)

Фильтр в Битриксе: «Дата начала: Текущий месяц» (или «Прошлый месяц» для предыдущего).
Снять со ВСЕХ колонок канбана число сделок.

В `<script>` файла три константы и массив:

- `TOTAL` = **Всего заявок/контактов** = сумма сделок ПО ВСЕМ колонкам канбана
  (включая «ОТВАЛИЛСЯ» и «Сотрудничество»).
- `MEET` = **Провёл встречу** = сумма колонок:
  `На презентации + Надо подумать + Моросят + Договор счёт + Сделка провалена + БОМЖИК ИКС 2 + Сделка успешна`
  (берём значения колонок как есть, в т.ч. колонку «Сделка успешна»).
- `WON` = **Успешных сделок** = число, которое называет заказчик ВРУЧНУЮ
  (НЕ значение колонки «Сделка успешна»). Например: июнь = 11, май = 3.
- `stages` — массив всех стадий с числами для таблицы «Распределение по стадиям».

**Конверсии (считаются в файле автоматически):**
- заявка → встреча = `MEET / TOTAL`
- встреча → сделка = `WON / MEET`
- общая воронка = `WON / TOTAL`

Полный список стадий канбана (слева направо):
НОВЫЕ · Вопросы · Созвониться · Раскачиваю · Договориться на видеовстречу ·
В работе сейчас · На презентации · Надо подумать · Моросят · Договор счёт ·
Сделка провалена · БОМЖИК ИКС 2 · Сделка успешна · ОТВАЛИЛСЯ · Сотрудничество · 2025.

## 6. Общая аналитика (`overview.html`)

Сводка по обоим источникам. При добавлении месяца — дописать его в массивы
`summary`, `avito`, `bitrix`, `link` (формат строки таблицы:
`['Показатель','значение_май','значение_июнь',...,'±динамика','good|bad|flat']`).
Ключевая связка: **стоимость успешной сделки = расходы Avito ÷ успешные сделки (WON)**.

## 9. Что прислать заказчику для нового месяца

Чтобы собрать июль, нужны:
1. **Avito:** скриншоты сводки и по каждому профилю (просмотры, контакты, избранное,
   расходы, активные объявления, динамика ±%).
2. **Bitrix:** скриншоты канбана сделок с фильтром по нужному месяцу (числа по всем колонкам).
3. **Вручную:** сколько клиентов реально подписалось/успешных сделок (значение `WON`).
